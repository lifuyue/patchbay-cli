# V6 Evaluation Workflow Productization Report
## Stage Summary
Decision: `proceed`. V6 productizes recommendation eval as `issue-finder eval recommendation`, with offline fixture reports and live six-profile skeleton reports generated by the CLI. The original test now reuses the production evaluator.
## Production Changes
- Added `src/recommendation/eval.rs` as the shared offline/live recommendation evaluation module.
- Added `issue-finder eval recommendation --offline --output <dir>` to write metrics.json, report.md, and visible.jsonl without network access.
- Added `issue-finder eval recommendation --live --refresh --limit 15 --output <dir>` to run the fixed six profiles with `record_exposure=false` and manual review placeholders.
- Live eval uses an isolated scoped home for each profile and cleans that temporary state after the profile run.
- Changed `tests/recommendation_eval.rs` to call the production evaluator and removed the old duplicated test support evaluator.
- Added an eval CLI contract test for offline report generation.
